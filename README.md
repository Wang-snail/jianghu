# 江湖

江湖是一个本地运行的 AI 数字组织生态系统。用户只提出目标，系统通过天机阁理解委托、组建临时帮派、调度弟子、分配功法、控制预算并沉淀经验。

它不是聊天机器人，也不是传统工作流工具。它把复杂任务包装成一次“委托江湖”的过程，让用户只关注目标、进展、风险和结果。

## 核心概念

- **天机阁**：用户直接对话和发起委托的位置，负责目标澄清、任务拆解、成立帮派、任命帮主和验收结果。
- **锦衣卫**：与天机阁平级的安全审计机构，负责识别规则规避、预算异常、输出风险、循环依赖和异常代理行为。
- **帮派**：围绕一次委托临时成立的项目组，任务结束后解散，经验和履历自动归档。
- **帮主**：当前帮派的项目负责人，负责招募弟子、领取功法、申请预算、监控交付和汇总结果。
- **弟子**：最小执行单元，职责单一、输入输出明确，按任务树路径协作。
- **客栈**：公共人才市场，只处理弟子、履历、岗位能力和任职关系。
- **藏经阁**：公共功法库，管理技能、模板、经验、SOP、提示词和安全审核后的修炼产物。
- **钱庄**：公共资源系统，管理铜钱、银两、金票、流水、预算、成本和效率积分。

## 本地运行

项目默认只在当前机器本地运行，不依赖外部托管。

```bash
npm install
npm run dev:room
```

默认访问地址：

- 本机：`http://127.0.0.1:4700/`
- 局域网手机：使用启动机器的局域网 IP，例如 `http://192.168.1.38:4700/`

如果需要让手机访问，启动时请绑定到局域网：

```bash
COMPANY_BIND_HOST=0.0.0.0 npm run dev:room
```

## AI 模型

系统优先使用当前环境中已经配置好的 MiMo、Claude Code、Codex 或 API 模型能力。MiMo 可用模型包括：

- `MiMo-V2.5-Pro`
- `MiMo-V2.5`
- `MiMo-V2-Pro`

不要使用不支持的 `mimo-v2-flash`。

## 运行流程

```text
用户提出目标
→ 天机阁澄清需求并生成需求文档
→ 用户确认
→ 天机阁成立临时帮派
→ 帮主任命弟子并分配功法
→ 钱庄拨付预算
→ 弟子执行子任务
→ 帮主整合结果
→ 天机阁验收
→ 锦衣卫审计
→ 输出结果
→ 履历、流水、经验归档
→ 帮派解散
```

## 资源与激励

钱庄使用三类内部资源，不与真实世界资金发生关联：

- **铜钱**：基础推理和普通文本生成。
- **银两**：工具调用、搜索、数据处理。
- **金票**：高成本模型、外部接口、长时间任务。

预算是任务级资源，用完即结算；声望是弟子级履历资产，跨任务积累。弟子的领域声望、稳定度、协作度、成本效率、近期表现会共同影响下次招募、预算信任度和可用功法等级。

## 防熵增规则

- 帮派只能围绕明确委托临时成立，任务结束自动解散。
- 弟子通讯只能沿任务树定义的路径传递。
- 输出必须符合藏经阁模板，格式不合规不能流向下游。
- 预算申请必须经过帮主，超支需天机阁审批或降级处理。
- 锦衣卫可对异常任务发出预警、暂停、复验或冻结异常声望增量。
- 修炼产物必须经过审核后才能进入藏经阁。

## 项目边界

江湖当前定位是本地 AI 组织系统。所有创建、关闭、修改、任务分派、信息传递、预算记录和经验沉淀都应通过本地接口完成。
