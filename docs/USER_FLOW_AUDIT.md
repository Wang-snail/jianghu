# 江湖用户流程检查

## 主路径

1. 打开 `http://localhost:4700/`。
2. 在天机阁提交委托。
3. 需求澄清代理通过对话补齐目标、边界、预算和交付格式。
4. 用户确认需求文档。
5. 天机阁成立临时帮派并任命帮主。
6. 帮主从客栈挑选弟子，从藏经阁领取功法，从钱庄申请预算。
7. 帮派执行，用户在江湖纵览查看全局状态，必要时进入帮派作战室。
8. 天机阁验收结果，锦衣卫检查风险。
9. 输出结果，沉淀履历、流水和功法经验，帮派解散。

## 页面检查点

- 天机阁：可对话、可发布委托、可进入需求澄清。
- 江湖纵览：每张卡片代表一个帮派，显示状态、当前动作、弟子标签、预算和沉淀结果。
- 客栈：只展示弟子、履历、能力和招募信息，不展示钱庄规则。
- 藏经阁：功法名称、分类和标签中文化，不显示来源字段。
- 钱庄：展示财气库、预算、流水、薪资、成本和效率积分。
- 锦衣卫：展示必要风险结论，不展示后台规则细节。
- 设置：保留必要本地配置，不出现旧通讯推荐、邀请链和无关说明。

## 验收标准

- 页面没有旧品牌词、旧端口说明和旧弹窗。
- 用户不需要理解内部技术名词即可发起委托。
- 帮派可启动、暂停、介绍并查看运行状态。
- 天机阁对话需要 AI 时能调用模型，不需要时走本地快捷动作。
- 任务完成后能够归档到弟子履历、钱庄流水和藏经阁经验。
