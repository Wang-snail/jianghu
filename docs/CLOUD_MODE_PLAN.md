# 本地运行边界

江湖当前只作为本地 AI 数字组织系统运行。本文件保留为架构边界说明，不再规划外部云托管模式。

## 原则

- 用户数据、帮派档案、弟子履历、钱庄流水和藏经阁功法默认保存在本机。
- 创建、关闭、修改、任务分派、预算结算和信息传递都通过本地接口完成。
- 页面不得引导用户购买、创建或连接外部托管运行环境。
- 手机访问通过局域网地址完成，例如 `http://192.168.1.38:4700/`。

## 本地访问

```bash
COMPANY_BIND_HOST=0.0.0.0 npm run dev:room
```

默认端口为 `4700`。如果机器 IP 是 `192.168.1.38`，同一局域网内的手机可访问：

```text
http://192.168.1.38:4700/
```

## 不再展示的内容

- 外部托管运行入口。
- 外部托管价格、套餐和购买流程。
- 邀请链、旧通讯推荐和非必要设备绑定。
- 面向开发者的后台治理规则。

## 后续如需扩展

任何远程访问或协作能力都应先转化为本地可控能力，并满足：

- 权限可审计。
- 数据可迁移。
- 任务可暂停和回滚。
- 不改变“用户只对天机阁发出委托”的产品模型。
